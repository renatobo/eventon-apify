# Release Notes Process

Create one file per release in this directory using the version number as the filename, for example:

- `release-notes/1.8.0.md`

Each file must include these top-level sections:

## New Features

- Describe user-visible additions in this release.

## Improvements

- Describe notable enhancements, polish, or documentation upgrades.

## Bug Fixes

- Describe defects resolved in the release.

The GitHub release workflow appends the full changelog comparison link automatically.
